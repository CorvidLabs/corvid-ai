# Context

corvid-ai already has four active canonical specs and a strict native Rust CI
job. It needs the organization-standard verified SDD lifecycle and unified
Trust gate without turning governance tools into crate dependencies or
replacing its independent Atlas Pages publication.

The existing companion files were generated placeholders. This migration
records stable requirement IDs and fills their context, tasks, and testing
plans from the behavior already defined in each canonical spec.
